Detailed Description

Oribatida authenticated encryption algorithm.

Oribatida is a family of authenticated encryption algorithms based on the SimP-256 and SimP-192 permutations which are built around reduced-round variants of the Simon-128-128 and Simon-96-96 block ciphers. There are two algorithms in the family:

  • Oribatida-256-64 with a 128-bit key, a 128-bit nonce, and a 128-bit tag, built around the SimP-256 permutation. This is the primary member of the family.
  • Oribatida-192-96 with a 128-bit key, a 64-bit nonce, and a 96-bit tag, built around the SimP-192 permutation.

References: https://www.isical.ac.in/~lightweight/oribatida/

Decrypts and authenticates a packet with Oribatida-192-96.

Parameters
mBuffer to receive the plaintext message on output.
mlenReceives the length of the plaintext message on output.
nsecSecret nonce - not used by this algorithm.
cBuffer that contains the ciphertext and authentication tag to decrypt.
clenLength of the input data in bytes, which includes the ciphertext and the 12 byte authentication tag.
adBuffer that contains associated data to authenticate along with the packet but which does not need to be encrypted.
adlenLength of the associated data in bytes.
npubPoints to the public nonce for the packet which must be 8 bytes in length.
kPoints to the 16 bytes of the key to use to decrypt the packet.
Returns
0 on success, -1 if the authentication tag was incorrect, or some other negative number if there was an error in the parameters.
