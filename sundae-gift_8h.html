Detailed Description

SUNDAE-GIFT encryption algorithm family.

The SUNDAE-GIFT family consists of several related algorithms:

  • SUNDAE-GIFT-0 with a 128-bit key, a 0-bit nonce, and 128-bit tag.
  • SUNDAE-GIFT-64 with a 128-bit key, a 64-bit nonce, and 128-bit tag.
  • SUNDAE-GIFT-96 with a 128-bit key, a 96-bit nonce, and 128-bit tag. This is the primary member of the family.
  • SUNDAE-GIFT-128 with a 128-bit key, a 128-bit nonce, and 128-bit tag.

SUNDAE-GIFT is resistant against nonce reuse as long as the combination of the associated data and plaintext is unique.

If a nonce is reused (or there is no nonce in the case of SUNDAE-GIFT-0), then two packets with the same associated data and plaintext will encrypt to the same ciphertext. This will leak that the same plaintext has been sent for a second time but will not reveal the plaintext itself.

Decrypts and authenticates a packet with SUNDAE-GIFT-0.

Parameters
mBuffer to receive the plaintext message on output.
mlenReceives the length of the plaintext message on output.
nsecSecret nonce - not used by this algorithm.
cBuffer that contains the ciphertext and authentication tag to decrypt.
clenLength of the input data in bytes, which includes the ciphertext and the 16 byte authentication tag.
adBuffer that contains associated data to authenticate along with the packet but which does not need to be encrypted.
adlenLength of the associated data in bytes.
npubPoints to the public nonce - not used by this algorithm.
kPoints to the 16 bytes of the key to use to decrypt the packet.
Returns
0 on success, -1 if the authentication tag was incorrect, or some other negative number if there was an error in the parameters.
